Algorithms & precise instructions

A computer can't think like you. It follows instructions exactly, so your instructions have to be clear and precise — this is the whole point of the "wacky monsters" activity.

Algorithm — a step-by-step series of instructions to solve a problem. Every algorithm needs three clear things:
1. Starting point
Where the algorithm begins.
2. Detailed instructions
Clear enough to follow with no extra explanation.
3. End point — where the algorithm stops.
💡
Why it matters: if instructions are vague ("add some legs"), everyone draws something different. Precise instructions ("add 4 legs to the bottom of the circle") get the same result every time — exactly how a computer needs them.

Drawing with Python turtle

The turtle module is a built-in set of Python commands that move a virtual pen around the screen. The turtle starts in the centre, facing right, at point (0,0).

🐢
If you name your turtle, every command starts with that name and a full stop: marker.fd(100). Naming is good practice once drawings get more complex.
03

Sequence

A sequence is a logical series of instructions carried out one after another, from start to finish. Each line runs in order until the program reaches the end.

# A square drawn as a sequence
import turtle
marker = turtle.Turtle()
marker.pencolor("green")
marker.fd(200)
marker.rt(90)
marker.fd(200)
marker.rt(90)
marker.fd(200)
marker.rt(90)
marker.fd(200)

Eight instructions, run top to bottom — nice, but a bit repetitive. That leads us to iteration…

04

Iteration (loops)

Iteration means using loops to repeat actions until a set point is reached. It's perfect for repeated patterns and makes code far more efficient.

# Same square, using a loop
import turtle
marker = turtle.Turtle()
marker.pencolor("blue")
for line in range(4):
    marker.fd(100)
    marker.lt(90)

The turning rule for any regular shape

Turn angle = 360 ÷ number of sides. Loop that many times.

# Triangle — 3 sides
for line in range(3):
    marker.fd(200)
    marker.lt(120)
# Hexagon — 6 sides
for line in range(6):
    marker.fd(100)
    marker.lt(60)
⚡
Why efficient code matters (Coding Golf!): fewer lines of code = uses less RAM, compiles quicker, uses less storage space, and is easier to debug. Blank lines, comments and import don't count as lines.

Bugs: logic vs syntax errors

  
Finding and fixing errors is called debugging (an error is a "bug"). There are two types you must be able to tell apart.


  

    
Logic error
      
A fault in the design of the program. It still runs, but produces the wrong output. Harder to spot because nothing crashes.

      
Example: a game multiplies the score instead of adding it. It works — just wrongly.

    

    
Syntax error
      
A mistake that breaks the language's rules and stops the program running. The computer shows an error message.

      
Example: spelling turtle wrong, missing quote marks, or missing indentation after for.

    

  


  
🔍
Debugging checklist: functions spelled correctly · text in sentence case · check characters & symbols · check indentation (especially after for and if) · proofread a printed copy.

Selection (IF statements)

  
Selection is where a condition triggers an action. The action only happens if the condition is met. In programming we use IF statements.

  

    IF … THEN … ELSE

    IF it is raining THEN put up umbrella ELSE leave it down.
  

  

    
answer = input("Is it raining? (yes/no) ")
if answer == "yes":
    print("Put up your umbrella!")
else:
    print("Leave it at home.")

  

  
Selection powers things like chatbots — the program chooses a reply based on what you type. Alan Turing's Turing Test asked whether a computer's replies could be mistaken for a human's.

Lists & random choice

  
A list stores many items together. Combined with the random module, you can pick an item at random — the trick behind the Magic 8-ball.

  

    
import random
responses = ["yes", "no", "maybe"]
question = input("Ask a yes/no question: ")
print(random.choice(responses))

  

  
🎱
A real Magic 8-ball has 20 responses — 10 positive, 5 negative, 5 neutral. The program picks one at random each time, so it looks like it's "thinking".

08
Subprograms (functions)

  
A subprogram — also called a subroutine, function or procedure — is a named block of code you can reuse. Breaking a big program into subprograms makes it shorter, simpler and easier to debug.

  

    
def boil_kettle():
    print("Fill kettle with water")
    print("Switch kettle on")
    print("Wait until boiled")

# Call the subprogram whenever you need it
boil_kettle()

  

  
A single instruction like "boil the kettle" hides lots of smaller steps — that's exactly the kind of thing a subprogram is for. It can also take a value: add_sugar(2).

String manipulation

  
A string is text. You can join, replace and slice strings — the basis of the Mad Libs word game, where the user's words fill blanks in a sentence.

  

    
noun = input("Give me a noun: ")
adjective = input("Give me an adjective: ")
print("The " + adjective + " " + noun + " ran away!")

  

  
📝
Type an adjective like "smelly" and a noun like "dragon" → "The smelly dragon ran away!" Change the words, change the story.

Key vocabulary

  
If you can define each of these, you're in good shape for the assessment.

  

    
algorithm
A step-by-step series of instructions with a clear start, detail and end.

    
sequence
Instructions carried out one after another, in order.

    
iteration
Using loops to repeat actions until a point is reached.

    
selection
Choosing an action based on a condition (IF statements).

    
counted loop
A loop that repeats a set number of times, e.g. range(4).

    
efficiency
Doing the same job in fewer lines of code.

    
RAM
Random access memory — efficient code uses less of it.

    
logic error
Program runs but gives the wrong output.

    
syntax error
Breaks the language rules; stops the program running.

    
debugging
Finding and fixing errors (bugs) in code.

    
list
A collection of items stored together.

    
subprogram
A reusable named block of code (function/procedure).

    
string
A piece of text, which can be joined or replaced.

    
Turing Test
A test of whether a computer's replies seem human.
